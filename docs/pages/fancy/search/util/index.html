package fancy.search.util

FancySearchClassNames

The following keys all expect string values, which will be added directly to the DOM as classes on the fancy search input element, the suggestion list and item elements, and various other buttons and components.

FancySearchKeyboardShortcuts

The following actions can be triggered via keyboard shortcuts. The provided Array of Ints for each action defines an array of keyboard keycodes that will cause the associated action to trigger.

FancySearchOptions

This is the object of options passed to the Search constructor.

FilterFunction

The filter function is given a function to convert from suggestion item to string (see SuggestionOptions.suggestionToString), the search string (value from the input), and a suggestion item to compare. It should return true if this item will be in the filtered list, and false to filter it out.

HighlightLetters

Given a function from suggestion item to string and a search string, the highlight letters function should return an array of Tuple2s. Each tuple marks the start index and length of a range that should be bold in the suggestion string that is printed (using the same toString method) in the list.

LiteralPosition

Possible positions for the literal "Search for: text" suggestion. First inserts it at the top of the list, and Last appends it at the bottom.

SelectionChooseFunction

This function will be called when a suggestion item is chosen (either by mouse click, or the enter key, etc). It is given the toString conversion function, the actual input element, and an Option of Some suggestion item or None.

SortSuggestions

The sort function is given a function from suggestion item to string, as well as the original search string. It is also given two suggestion items, and it should return -1, 0, or 1, similar to most array order-by functions.

SuggestionOptions

Suggestion options are passed to the Search constructor as a child of FancySearchOptions. These options control the look and behavior of the dropdown suggestion list.
